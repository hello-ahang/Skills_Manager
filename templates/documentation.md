# 文档规范

## 代码注释
- 所有公共 API 必须有 JSDoc/docstring
- 复杂逻辑需要行内注释
- TODO 注释必须关联 Issue 编号
- 避免注释显而易见的代码

## README 要求
- 项目简介和功能概述
- 快速开始指南（安装、配置、运行）
- API 文档链接
- 贡献指南
- License 信息

## 变更日志
- 遵循 Conventional Commits
- 每次发布更新 CHANGELOG.md
- 分类: Added / Changed / Deprecated / Removed / Fixed / Security

## API 文档
- 使用 OpenAPI/Swagger 规范
- 包含请求/响应示例
- 标注必填/可选参数
- 列出所有可能的错误码

## 项目特定要求
- 使用中文编写用户文档
- 使用英文编写代码注释
- 架构决策记录 (ADR) 使用 Markdown
