# 测试策略

## 测试类型
- 单元测试: 覆盖所有业务逻辑
- 集成测试: 覆盖 API 端点
- E2E 测试: 覆盖关键用户流程

## 测试规范
- 测试文件命名: `*.test.ts` 或 `*.spec.ts`
- 使用 describe/it 组织测试
- 每个测试只验证一个行为
- 使用 AAA 模式 (Arrange, Act, Assert)

## 覆盖率要求
- 最低覆盖率: 80%
- 关键路径: 100%
- 新代码必须有对应测试

## Mock 策略
- 外部 API 调用必须 Mock
- 数据库操作使用内存数据库
- 时间相关使用 fake timers
- 文件系统使用 memfs 或 mock

## 测试数据
- 使用 factory 模式生成测试数据
- 避免硬编码测试数据
- 敏感数据使用 faker 生成
