# 架构设计规范

## 项目结构
```
src/
├── components/    # UI 组件
├── pages/         # 页面组件
├── stores/        # 状态管理
├── hooks/         # 自定义 Hooks
├── api/           # API 调用
├── types/         # 类型定义
└── lib/           # 工具函数
```

## 设计原则
- 单一职责原则: 每个模块只负责一个功能
- 依赖倒置原则: 依赖抽象而非具体实现
- 接口隔离原则: 接口应该小而专注
- 开闭原则: 对扩展开放，对修改关闭

## 分层架构
- 表现层: UI 组件和页面
- 业务层: 业务逻辑和状态管理
- 数据层: API 调用和数据持久化

## 组件设计
- 容器组件 vs 展示组件分离
- 使用组合而非继承
- Props 接口明确定义
- 避免过深的组件嵌套（最多 3-4 层）

## 错误处理
- 统一错误边界 (Error Boundary)
- 分级错误处理（组件级 → 页面级 → 应用级）
- 用户友好的错误提示
- 错误日志上报

## 性能要求
- 首屏加载 < 3s
- API 响应 < 500ms
- 组件懒加载 (React.lazy)
- 列表虚拟化（大数据量时）
- 图片懒加载
